# Phase quick Plan 260319-m88: Dead Code Removal and Minor Cleanup Summary

**One-liner:** Removed Phase 1 spike files and unused export, fixed invalid CSS property, eliminated redundant array filter pass in scoreEyeContact.

## What Was Built

No new features — pure cleanup and correctness fixes:

1. **Deleted src/spikes/ directory** (4 files, ~333 lines): SpikeRunner.tsx, mediapipe-spike.ts, speech-spike.ts, webm-spike.ts. These were Phase 1 exploration files used for manual verification of MediaPipe, WebM, and filler detection viability. They were never imported by production code.

2. **Removed `getNearbyEvents` from eventSync.ts**: This function was exported but never imported anywhere in the codebase. Only `getNearestEvent` (used by Timeline.tsx) is kept.

3. **Fixed invalid CSS property in RecordingScreen.tsx**: Removed `tabularNums: 'tabular-nums'` from the timer div's inline style object. This is not a valid CSS property name. The correct `fontVariantNumeric: 'tabular-nums'` was already present on the next line, and `className="tabular-nums"` provides it via Tailwind as well.

4. **Eliminated redundant filter pass in scoreEyeContact (scorer.ts)**: The function was filtering `events` twice — once for `eye_contact_break` only (to count breaks), and once for both `eye_contact_break | eye_contact_resume` (to compute away time). Refactored to a single sorted `contactEvents` array; `breakCount` is derived from it with a secondary filter. Behavior is identical.
