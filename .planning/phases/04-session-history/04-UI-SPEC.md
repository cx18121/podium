## Interaction Contract

### Session List
- Renders metadata only — `videoBlob` NOT loaded until user opens the session (lazy load on HIST-02 navigation)
- Sessions ordered by `createdAt` descending (most recent first)
- Empty state renders immediately with heading + CTA (no spinner)
- Click row → transition to 'review' view with selected `sessionId`
- Delete flow → show `DeleteConfirmModal` → on confirm remove from Dexie → row disappears from live query

### Sparkline Charts
- Data source: `db.sessions.orderBy('createdAt').limit(10).toArray()` — last 10 sessions only
- Each chart renders per-dimension scores across those sessions (x = session index, y = dimension score 0–100)
- If fewer than 2 sessions exist: render chart area with `text-gray-600 text-xs` message "Record more sessions to see trends"
- No interactivity on sparklines (Phase 4 scope — hover tooltips deferred to v2)

### Navigation: App.tsx state machine extension
- New view: `'history'`
- History accessible from: SetupScreen (returning user flow — "View History" link) and Review page (back button)
- From history, "Record New Session" CTA transitions to `'setup'`

### Storage Quota
- Read via `navigator.storage.estimate()` on mount
- No polling — one-time read on HistoryView mount
- If API unavailable: hide the bar entirely (no error state shown)
