      posts { type: 'ready' } on success or { type: 'error', message } on failure
    - 'frame' message: receives { type: 'frame', bitmap, timestampMs }; if any model is null,
      closes bitmap and returns; runs detectForVideo/recognizeForVideo using performance.now()
      as the actual timestamp (NOT e.data.timestampMs — avoids clock skew); calls bitmap.close()
      in try/finally; derives events via deriveEvents() stub (returns [] for now — populated in 02-02);
      pushes derived events to pendingEvents; posts { type: 'frame_ack' }
    - 'stop' message: posts { type: 'events', events: [...pendingEvents] }; resets pendingEvents = []
    - 'cleanup' message: closes all three models, nulls them, posts { type: 'cleanup_done' }
    - busy flag: set a module-level `let busy = false` flag; in the 'frame' handler, if busy===true,
      close the bitmap and return without queuing (prevents frame backup per Pitfall 4 in RESEARCH.md)
  </behavior>

    Critical implementation notes (all from RESEARCH.md pitfalls section):
    1. FaceLandmarker MUST be initialized with outputFaceBlendshapes: true — the default is false

    Implementation details for useMLWorker:
    - workerRef: useRef<Worker | null>(null)
    - frameIntervalRef: useRef<ReturnType<typeof setInterval> | null>(null)
    - startWorker(videoEl): creates new Worker(workerUrl, { type: 'classic' }), sends { type: 'init' },
      waits for { type: 'ready' } message (Promise), then starts frame pump:
      setInterval(async () => {
        if (videoEl.readyState < 2) return;
        const bitmap = await createImageBitmap(videoEl);
        workerRef.current?.postMessage({ type: 'frame', bitmap, timestampMs: performance.now() }, [bitmap]);
      }, 150)
    - stopWorker(): clears the frame interval, sends { type: 'stop' }, returns Promise<SessionEvent[]>
      that resolves when worker posts { type: 'events', events }
    - cleanupWorker(): sends { type: 'cleanup' }, terminates worker after cleanup_done or after 500ms timeout
