## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding (input border offsets) |
| sm | 8px | Compact button padding, gap between small sibling elements |
| md | 16px | Default element spacing, input horizontal padding (`px-4`) |
| lg | 24px | Section padding, card gap (`gap-6`) |
| xl | 32px | Layout gaps between major sections (`gap-8`) |
| 2xl | 48px | Large vertical rhythm between score rows (`gap-12`) |
| 3xl | 64px | Page-level padding on review screen (max-w container centering) |

Exceptions:
- Timeline marker dot: 12px diameter (`w-3 h-3`) — off-scale but required for hit-target sizing; do not change to 8px or 16px
- Touch target minimum: 44px height on marker buttons — apply `min-h-[44px]` on the marker wrapper element; do not use `py-2.5` (not a multiple of 4)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 400 (regular) | 1.4 |
| Heading | 24px (`text-2xl`) | 700 (bold) | 1.2 |
| Display | 32px (`text-3xl`) | 700 (bold) | 1.2 |

**Usage rules:**
- Session title on review screen: Heading (24px bold)
- Dimension names in scorecard (Eye Contact, Fillers, etc.): Label (14px regular)
- Scores (e.g. "82 / 100"): Label (14px regular), or Heading (24px bold) for overall score only
- Supporting detail strings (e.g. "3 breaks", "~2.1/min"): Label (14px regular), `text-gray-400`
- Video metadata line (duration · date): Label (14px regular), `text-gray-400`
- "Record Another Session" button: 16px bold (weight 700, `font-bold`) — red on dark background; bold at 16px provides sufficient visual emphasis

**Source:** Extracted from existing components. `text-4xl font-bold` used on `Home.tsx` headline only (out of scope for Phase 3). Phase 3 uses `text-2xl`/`text-3xl` max.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `gray-950` (`#030712`) | Page background (`min-h-screen bg-gray-950`), video player background |
| Secondary (30%) | `gray-800` (`#1f2937`) / `gray-900` (`#111827`) | Score cards/rows, timeline bar background, input/modal backgrounds |
| Accent (10%) | `red-600` (`#dc2626`) / `red-500` (`#ef4444` on hover) | See accent reserved-for list below |
| Destructive | `red-600` / `red-500 hover` | "Record Another Session" button — the only destructive-adjacent action; no delete/permanent-loss actions in Phase 3 |

**Accent reserved for:**
1. Primary CTA button: "Record Another Session" (`bg-red-600 hover:bg-red-500`)
2. Timeline event marker dots (`bg-amber-400`) — amber is the marker-specific accent; distinct from CTA red to signal "event dot, not interactive button"
3. Score bar fill for dimension scores — use `red-600` fill on a `gray-700` background track
4. Focus ring on interactive elements (`focus:border-red-500`, `focus:ring-red-500`)

**Amber marker accent:**
- `amber-400` (`#fbbf24`) is reserved exclusively for timeline event marker dots
- `amber-300` for focus ring on hovered/focused marker buttons
- Do not reuse amber for any other element in Phase 3

**Text colors:**
- Primary text: `text-white`
- Secondary text: `text-gray-400`
- Muted/caption: `text-gray-500` / `text-gray-600`
- Error text: `text-red-400` (already used in existing `Review.tsx` error state)

**Source:** Color palette extracted from all existing components. Amber introduced for timeline markers per RESEARCH.md `Timeline.tsx` pattern (line: `className="... bg-amber-400 ..."`).

## Component Inventory

The following new components are required for Phase 3. Existing components are listed for reference only.

### New: ScorecardView

**File:** `src/components/ScorecardView/ScorecardView.tsx`
**Props:** `scorecard: ScorecardResult` (from `aggregateScores`)
**Layout:**
- Overall score: large centered number (Display size, bold), label "Overall Score" beneath it (Label size, `text-gray-400`)
- Per-dimension rows: 5 rows, each showing dimension name + score bar + detail text
- Score bar: `<div>` with `bg-gray-700` background track, inner `<div>` with `bg-red-600` fill at `width: {score}%`, `h-2 rounded-full`
- No border between rows; `gap-4` between each row
- Container: `bg-gray-900 rounded-2xl p-6` card, `w-full max-w-2xl` constrained

**States:**
- Loading: show `text-gray-400` "Calculating scores..." placeholder (spinner not required)
- No-data dimension: score bar renders at `width: 50%` with `text-gray-500` detail showing "Insufficient data"
- Normal: all five dimension rows render with score bar and detail text

### New: AnnotatedPlayer

**File:** `src/components/AnnotatedPlayer/AnnotatedPlayer.tsx`
**Props:** `videoUrl: string`, `durationMs: number`, `events: SessionEvent[]`
**Layout:**
- `<video>` element: `w-full max-w-2xl rounded-xl bg-black`, no native `controls` attribute (replaced by custom timeline)
- Custom play/pause: single centered play button overlaid on video using absolute positioning, `opacity-0 group-hover:opacity-100 transition-opacity`; do NOT add a full control bar (out of scope for Phase 3 — native controls only toggled off, timeline click-to-seek is the primary interaction)
- Timeline below video: separate `<div>` (`h-8 rounded-full bg-gray-800`) — NOT overlaid on `<video>`
- Container: `flex flex-col gap-2 w-full max-w-2xl`

**Note on play/pause:** Phase 3 ships with the `<video>` element's native controls disabled and replaced by: (1) clicking anywhere on the video element to toggle play/pause, and (2) the custom timeline below. No full custom control bar (progress, volume, fullscreen) is in scope.

## Interaction Contract

### Video playback
- Clicking anywhere on the `<video>` element toggles play/pause (`video.paused ? video.play() : video.pause()`)
- No keyboard shortcut mapping required for Phase 3
- `aria-label="Session playback"` — existing attribute; keep

### Timeline seek
- Clicking the timeline bar (not a marker): seek to click fraction × durationMs
- Clicking a marker dot: seek to `event.timestampMs`; `e.stopPropagation()` required to prevent also triggering bar click
- Marker dots: `title` attribute for browser-native tooltip; `aria-label` matches `title` for accessibility

### Score bar
- Static — no hover, no click, no tooltip
- `role="meter"` on inner fill div, `aria-valuenow={score}`, `aria-valuemin={0}`, `aria-valuemax={100}`, `aria-label="{dimension} score"`

### Transitions
- All interactive elements: `transition-colors` (200ms, matches existing components)
- Marker dot hover: `hover:scale-150 transition-transform` (matches RESEARCH.md pattern)
- No page-level transitions between views (existing App.tsx pattern has none)

## Accessibility Contract

- `<video>` element: `aria-label="Session playback"`
- Marker buttons: `aria-label={eventLabel(event)}` (same as `title`)
- Score bars: `role="meter"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label`
- Overall score number: wrapped in `<output>` element with `aria-label="Overall score"`
- Error states: `role="alert"` on error message container
- Loading states: `aria-busy="true"` on loading container
- Focus order: title → metadata → scorecard card → video player → timeline → CTA button
