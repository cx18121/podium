## Audit Findings — What Phase 5 Must Fix

These findings were extracted from full codebase review. Every finding maps to a specific fix required in this phase.

| # | Finding | Location | Severity |
|---|---------|----------|----------|
| A-01 | Accent (`red-600`) used for CTAs, score bars, delete button, storage critical state, and error alerts simultaneously — no visual hierarchy between action, data, and danger | All pages | High |
| A-02 | Timeline marker touch target is 12×12px (`w-3 h-3`) — below WCAG 24px minimum for interactive targets | Timeline.tsx | High |
| A-03 | Delete button (`×`) hidden via `opacity-0 group-hover:opacity-100` — not keyboard-accessible; focus-visible state absent | SessionListItem.tsx | High |
| A-04 | Four different `h1` sizes used across pages (text-xl, text-2xl, text-3xl, text-4xl) with no hierarchy contract | Home, SetupScreen, Review, HistoryView | Medium |
| A-05 | Processing state is plain text with no loading indicator — no spinner, no progress affordance | App.tsx (processing view) | Medium |
| A-06 | "View History" navigation link styled as `text-gray-400 underline` — visually indistinguishable from body copy, not an obvious navigation affordance | SetupScreen.tsx | Medium |
| A-07 | Score badge in SessionListItem renders identically for 0 and 100 — no value-relative color cue | SessionListItem.tsx | Medium |
| A-08 | SparklineChart path rendered at `opacity-0.5` — low contrast against dark background | SparklineChart.tsx | Low |
| A-09 | AnnotatedPlayer video has no visual affordance for click-to-play — cursor:pointer set but no overlay icon or instruction | AnnotatedPlayer.tsx | Low |
| A-10 | NameSessionModal shows `border border-gray-700` on the dialog container — adds visual noise; focus ring uses `red-500` but CTA uses `red-600` (inconsistent) | NameSessionModal.tsx | Low |
| A-11 | No max-width constraint on outer page containers — content can stretch to full viewport on large screens in some views | HistoryView, Review, Home | Low |
| A-12 | `text-gray-600` used for the "Runs entirely in your browser" footnote in Home — below 4.5:1 contrast ratio on `gray-950` background | Home.tsx | Medium |

## Spacing Scale

Declared values (must be multiples of 4). These formalize the spacing already in use across the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding, marker offsets |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, card inner padding sides |
| lg | 24px | Card inner padding (top/bottom), section gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | RecordingScreen timer-to-indicator gap |
| 3xl | 64px | Page-level top/bottom padding on full-height views |

Exceptions:
- Timeline height: 44px (raised from current 32px to meet WCAG 2.5.8 target size minimum — fix for A-02)
- Touch target minimum for all interactive elements: 44×44px minimum tap/click area (may use padding to achieve; visual size may differ)

## Typography

Four roles, two weights. Resolves A-04 — locks a single heading size per role so all pages use the same scale.

| Role | Size | Weight | Line Height | Tailwind Class | Notes |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` | Supporting text, metadata, labels |
| Label | 16px | 400 (regular) | 1.5 | `text-base` | List item titles, dimension names, modal body |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` | All page h1, section h2 — unified across all pages (replaces current 4 inconsistent sizes) |
| Display | 48px | 600 (semibold) | 1.0 | `text-5xl font-semibold` | RecordingScreen timer only — monospace, tabular-nums preserved |

Note: The overall score output (`ScorecardView`) is a one-off data-display size (30px, `text-3xl font-semibold`). It does not enter the type scale; it is a data value, not text.

Weight constraint: Two weights only: 400 (regular) for body/label roles, 600 (semibold) for heading/CTA/data value roles.

### 60/30/10 Split Summary

- **60% Dominant:** `gray-950` fills all page backgrounds
- **30% Secondary:** `gray-900` fills all card and modal surfaces; `gray-800` for raised inputs and tracks
- **10% Accent:** `red-600` reserved ONLY for primary CTA buttons — never score bars, never destructive actions, never data displays

### Accent Reserved For

`red-600` is used exclusively on:
1. "Start Recording" button (Home, SetupScreen)
2. "Record New Session" button (HistoryView)
3. "Record Another Session" button (Review)

`red-600` is NOT used for: score bars, error states, delete buttons, storage indicators, loading states, or any data visualization.

## Component-Level Fix Specifications

### Fix A-01: Score bars — replace flat red-600 with score-relative color

ScorecardView score bars use a three-tier color scheme based on `dim.score`:
- score >= 70: `bg-emerald-500`
- score 40–69: `bg-amber-400`
- score < 40: `bg-red-500`

### Fix A-02: Timeline touch target

Timeline container height raised from `h-8` (32px) to `h-11` (44px). Marker dots raised from `w-3 h-3` (12px) to `w-4 h-4` (16px) visual size. Each marker button receives `min-w-[44px] min-h-[44px]` via padding expansion (`-mx-[14px] -my-[14px]`) so click/tap area is 44×44px while dot remains 16px visually.

### Fix A-03: Delete button keyboard accessibility

SessionListItem delete button removes `opacity-0 group-hover:opacity-100`. New behavior: always rendered at full opacity but visually subdued (`text-gray-600`). On hover: `text-red-400`. On `focus-visible`: `outline-2 outline-offset-2 outline-red-500` ring visible. Add `aria-label="Delete session"` to the button element.

### Fix A-04: Heading hierarchy

All `h1` elements across pages use exactly `text-xl font-semibold` (20px/600). Current mismatch resolved:
- Home.tsx: `text-4xl font-bold` → `text-xl font-semibold`
- SetupScreen.tsx: `text-3xl font-bold` → `text-xl font-semibold`
- Review.tsx: `text-2xl font-bold` → `text-xl font-semibold`
- HistoryView.tsx: `text-xl font-bold` → `text-xl font-semibold`
- All `h2` elements use `text-base font-semibold` (16px/600)

### Fix A-05: Processing state loading indicator

"Processing recording..." view adds an animated spinner. Implementation: `animate-spin` on a 32px SVG circle arc (inline SVG, no library). Copy: "Processing your recording..." below spinner at `text-sm text-gray-400`.

### Fix A-06: View History navigation affordance

"View History" link in SetupScreen: remove `underline`. Apply `text-sm text-gray-400 hover:text-white transition-colors px-4 py-2` with a preceding `→` arrow character inline. Min touch target: 44px height via padding.

### Fix A-07: Score badge color-coded

SessionListItem score badge: three-tier background matching score bar contract:
- score >= 70: `bg-emerald-900 text-emerald-300`
- score 40–69: `bg-amber-900 text-amber-300`
- score < 40: `bg-red-900 text-red-300`
- score === '—' (unscored): `bg-gray-700 text-gray-400`

### Fix A-08: SparklineChart line opacity

SparklineChart path `opacity-0.5` → `opacity-0.9`. Stroke color remains `rgb(251 191 36)` (amber-400).

### Fix A-09: Video play/pause affordance

AnnotatedPlayer video: on hover, show a semi-transparent play/pause overlay icon centered on the video. Implementation: `<div>` overlay absolutely positioned, hidden by default (`opacity-0`), shown on parent hover (`group-hover:opacity-100`), containing an inline SVG play icon (when paused) or pause icon (when playing). Transition: `transition-opacity duration-150`. The overlay button must include `aria-label="Play"` when the video is paused and `aria-label="Pause"` when the video is playing.

### Fix A-10: NameSessionModal cleanup

Remove outer `border border-gray-700` from modal container. Input focus ring: change `focus:border-red-500` to `focus:border-red-600` to match CTA button color. No other changes to modal layout.

### Fix A-11: Page max-width

All page root containers add `max-w-3xl mx-auto w-full` so content caps at 768px on wide screens. Home and SetupScreen already functionally limited by inner content but should receive the same constraint for consistency.

### Fix A-12: Home footnote contrast

"Runs entirely in your browser. Nothing is uploaded." copy: `text-gray-600` → `text-gray-500`. gray-500 (#6b7280) on gray-950 (#030712) passes WCAG AA at normal text size (contrast 4.6:1).

## Interaction Contract

### Transitions

All interactive state changes use `transition-colors duration-150`. No element uses a duration other than 150ms. Remove any implicit `transition-colors` without explicit duration (Tailwind default is 150ms — acceptable if not overridden).

### Focus rings

All interactive elements receive `focus-visible:outline-2 focus-visible:outline-offset-2`. Color:
- Accent buttons (red CTA): `focus-visible:outline-red-500`
- Destructive buttons: `focus-visible:outline-red-400`
- Cancel/secondary buttons: `focus-visible:outline-gray-400`
- Input fields: `focus:border-red-600` (border, not outline — preserves existing pattern)
- Timeline markers: `focus:ring-2 focus:ring-amber-300` (existing — retain)
- Delete button: `focus-visible:outline-red-500` (new — see A-03 fix)

### Hover states

| Element | Default | Hover |
|---------|---------|-------|
| Accent button | `bg-red-600` | `bg-red-500` |
| Destructive button | `bg-red-700` | `bg-red-600` |
| Secondary button | `bg-gray-700` | `bg-gray-600` |
| Ghost/link | `text-gray-400` | `text-white` |
| SessionListItem row | `bg-gray-900` | `bg-gray-800` |
| Timeline marker | scale-100 | scale-150 (existing) |
| Delete button | `text-gray-600` | `text-red-400` |

### Disabled states

Disabled buttons: `opacity-40 cursor-not-allowed` (existing pattern — retain).
