# Phase 7 — UI Design Contract: Visual Redesign

> Visual and interaction contract for the full-app visual redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Objective:** Replace the generic flat-gray-950 + red-600 aesthetic with a distinctive, premium coaching-tool identity. The design must feel purposeful and authoritative — like a tool a professional would pay for — not a demo or side project.

**Design direction:** Dark base with deep blue-slate tones, a single high-saturation accent (electric blue/indigo), tight typographic hierarchy using a variable-weight display font, layered depth via subtle borders and glows, and data visualizations (scorecard bars, sparklines, storage bar) treated as first-class design elements.

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline tight spacing |
| sm | 8px | Compact element spacing, inner card padding rows |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, vertical page rhythm |

Exceptions:
- Touch targets: minimum 44px per axis (maintained from Phase 05 via padding expansion)
- Timeline scrubber hit area: 48px tall (8px visual bar + padding)
- Score ring / large display numbers: 80px container to breathe against surrounding content

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 48px | 600 (semibold) | 1.1 | Overall score number in scorecard ring |
| Heading | 20px | 600 (semibold) | 1.2 | Page h1, section headings, modal titles |
| Body | 16px | 400 (regular) | 1.5 | Paragraph copy, list items, descriptions |
| Label | 13px | 400 (regular) | 1.4 | Dimension labels, timestamps, captions, badge text, nav secondary |

**Weights in use: 400 and 600 only** — Display uses 600 (size alone at 48px provides visual dominance; 600 vs 700 is imperceptible). Label uses 400 with `letter-spacing: 0.02em` which compensates for small-size legibility.

**Font features:**
- Timers and score numbers: `font-variant-numeric: tabular-nums` — prevents layout shift as digits change
- Headings: `letter-spacing: -0.02em` (tracking-tight) — tightens large text for polish
- Labels: `letter-spacing: 0.02em` (tracking-wide) — improves legibility at small size

**Source:** Phase 05-04 normalized all h1 to `text-xl font-semibold` (20px/600). Display and Label tiers are new additions for the redesign.

## Depth and Layering Contract

This section is new for Phase 7 — it defines the visual depth system that gives the design its premium feel.

| Layer | Background | Border | Shadow/Glow |
|-------|------------|--------|-------------|
| Page | `#080c14` | none | none |
| Card | `#111827` | `rgba(255,255,255,0.07)` 1px | none |
| Card (hover) | `#1a2235` | `rgba(255,255,255,0.12)` 1px | none |
| Modal overlay | `rgba(0,0,0,0.7)` | — | — |
| Modal panel | `#111827` | `rgba(255,255,255,0.10)` 1px | `0 25px 50px rgba(0,0,0,0.6)` |
| Score ring (outer) | transparent | `#6366f1` 2px | `0 0 24px rgba(99,102,241,0.3)` |
| CTA button | `#6366f1` | none | `0 0 20px rgba(99,102,241,0.25)` on hover |

**Rule:** Depth is created by border opacity and controlled glow, not by box-shadow stacking. Maximum one glow per element. No drop shadows on flat cards — borders only.

## Implementation Notes for Executor

1. **Color tokens as CSS custom properties:** Define all colors in `src/index.css` as `@layer base { :root { --color-bg: #080c14; ... } }` so they are reusable without Tailwind config changes (Tailwind v4 arbitrary value fallback).

2. **Tailwind v4 arbitrary values:** Use `bg-[#080c14]`, `border-[rgba(255,255,255,0.07)]` etc. — no tailwind.config.js needed.

3. **SVG score ring:** Implement as an inline SVG `<circle>` with `stroke-dasharray` driven by the score percentage. Keep it self-contained in ScorecardView — no external SVG library.

4. **Sparkline:** Already implemented as SparklineChart component. Update stroke color to `#6366f1` (indigo-500). No structural changes needed.

5. **Font loading:** Add `<link rel="preconnect" href="https://fonts.googleapis.com">` and Inter font import to `index.html`. Apply `font-family: 'Inter', system-ui, sans-serif` in `src/index.css` body rule.

6. **`tabular-nums`:** Apply `font-variant-numeric: tabular-nums` via Tailwind class `tabular-nums` to all score numbers, timer displays, and duration fields.

7. **`motion-safe:`:** Wrap all transition classes: `motion-safe:transition-colors motion-safe:duration-150` — bare `transition-colors` is fine if project Tailwind config already sets `prefers-reduced-motion` to 0 animations.

8. **max-w-3xl mx-auto:** Already applied to all pages per Phase 05-04. Maintain across all screens.
